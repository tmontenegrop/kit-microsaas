# kit-microsaas — Context for Auditing LLMs

**Updated:** 2026-06-30

## Philosophy

kit-microsaas is NOT a SaaS platform. It is a **MicroSaaS kit**: a collection of tiny, single-purpose tools that anonymous users pay for per-use.

### Core beliefs driving every design decision:

1. **Anonymous-first.** No login, no registration, no password. The user owes us nothing. We prove value first, ask for money only at the moment of download.
2. **Pay-per-event + free trial.** Not subscriptions. The user gets 30 free docs per IP per 30 days. After that: Batch ($2.990 per generation) or Pase 30 días ($6.990 unlimited). This lowers the barrier to zero. If the tool is useless, the user loses nothing. If it works, they pay once.
3. **No support, no onboarding, no retention.** There is no dashboard, no user profile, no email, no "my documents" page. The transaction is: upload → pay → download → goodbye. This is intentional and non-negotiable.
4. **Micro-services = micro-code.** Each tool should be ~200–500 lines of Go. If a tool needs more complexity, it is not a micro-tool. Millennial developers over-engineer; this kit fights that.
5. **Built by an LLM-first engineer.** The sole developer of this kit does not write Go by hand. He writes prompts. Every line of code here was generated by an LLM. The code must be simple enough that an LLM can refactor it in one shot, and robust enough that an LLM audit catches flaws.
6. **B2B micro-PYME Chile.** The target is accountants, HR admins, and small business administrators in Chile. They use iLovePDF, not Salesforce. They pay with Flow.cl (the dominant Chilean payment gateway), not Stripe. They do not have company credit cards.

## Schema (SQLite)

```sql
CREATE TABLE rate_limits (
    key         TEXT PRIMARY KEY,
    attempts    INTEGER NOT NULL DEFAULT 0,
    expires_at  TEXT NOT NULL
);

CREATE TABLE tools (
    id          TEXT PRIMARY KEY,
    slug        TEXT UNIQUE NOT NULL,
    name        TEXT NOT NULL,
    description TEXT NOT NULL DEFAULT '',
    price_clp   INTEGER NOT NULL DEFAULT 2990,
    active      INTEGER NOT NULL DEFAULT 1,
    created_at  TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE downloads (
    id                TEXT PRIMARY KEY,
    tool_id           TEXT NOT NULL REFERENCES tools(id),
    token             TEXT UNIQUE NOT NULL,
    token_hash        TEXT NOT NULL DEFAULT '',
    status            TEXT NOT NULL DEFAULT 'draft', -- draft | ready | paid | expired
    ip_address        TEXT NOT NULL DEFAULT '',
    created_at        TEXT NOT NULL DEFAULT (datetime('now')),
    paid_at           TEXT,
    expires_at        TEXT NOT NULL,
    file_path         TEXT NOT NULL DEFAULT '',      -- path to template .docx (draft) or result ZIP (paid)
    markers           TEXT NOT NULL DEFAULT '[]',    -- JSON array of detected {{markers}}
    file_name_markers TEXT NOT NULL DEFAULT '[]',    -- JSON array of markers used for filenames
    data_rows         TEXT                           -- JSON array of parsed Excel rows
);

CREATE TABLE payments (
    id              TEXT PRIMARY KEY,
    download_id     TEXT NOT NULL REFERENCES downloads(id),
    amount          INTEGER NOT NULL,
    status          TEXT NOT NULL DEFAULT 'pending',  -- pending | confirmed | failed
    payment_token   TEXT,
    flow_token      TEXT,
    flow_order      TEXT,
    created_at      TEXT NOT NULL DEFAULT (datetime('now')),
    confirmed_at    TEXT,
    error_message   TEXT
);

CREATE TABLE idempotency_keys (
    key         TEXT PRIMARY KEY,
    response    TEXT,
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    expires_at  TEXT
);

CREATE TABLE trial_tracking (
    key         TEXT PRIMARY KEY,
    doc_count   INTEGER NOT NULL DEFAULT 0,
    window_start TEXT NOT NULL DEFAULT (datetime('now')),
    expires_at  TEXT
);
```

## Security model (audit checklist)

| Concern | Implementation |
|---------|---------------|
| **CSRF** | Double Submit Cookie. HTMX sends `X-CSRF-Token` header via `hx-headers`. Exempt: `/webhook/flow`. |
| **XSS** | Go `html/template` auto-escapes. CSP allows only `'self'` + `unpkg.com` (HTMX). Tailwind compiled statically, no `unsafe-inline`. |
| **SQL injection** | All queries use parameterized placeholders (`?`). No string concatenation. |
| **Path traversal** | `storage.resolve()` normalizes path and checks `filepath.Rel` does not start with `..`. |
| **File upload** | `SaveWithValidation()` reads into memory, checks `http.DetectContentType` against whitelist, enforces 10MB limit. |
| **Rate limiting** | Per `upload:{IP}`, `data:{IP}`, `pay:{IP}` keys in SQLite. 10 req/h per resource. `ratelimit.Check()` called in `Upload`, `DataUpload`, `Pay`. Cookie `device_id` set on first upload. |
| **Download token** | 64 random hex chars from `crypto/rand`. Stored hashed (SHA-256) as `token_hash`. Never exposed in logs. 1h TTL (extendable). |
| **Webhook HMAC** | Flow.cl signs requests with shared secret. `VerifyHMAC()` validates SHA-256 HMAC before processing. |
| **Error leakage** | In production, all errors return generic "Error interno del servidor". Full error logged server-side only. |
| **Security headers** | CSP (no `unsafe-inline`, no CDN script sources), X-Frame-Options: DENY, X-Content-Type-Options: nosniff, HSTS (1 year, preload), Referrer-Policy, Permissions-Policy. |
| **HTTPS** | Production only: redirects all HTTP to HTTPS (301). |
| **CORS** | Configurable via `ALLOWED_ORIGINS` env var. Disabled by default (empty = no CORS header). |
| **Cleanup** | Goroutine every 1h: expired downloads (draft/ready >24h) → status='expired', deletes upload files. |

## Bugs corregidos

1. **CSP bloqueaba HTMX** — `script-src` no incluía `unpkg.com`, el navegador bloqueaba la carga de HTMX. Fix: agregar `https://unpkg.com` a `script-src`.
2. **hx-vals enviaba JSON, handler esperaba form-urlencoded** — `hx-vals='{"marker": "{{.Name}}"}'` enviaba JSON pero `ToggleFileNameMarker` usa `r.FormValue("marker")` (form-urlencoded). Fix: `<form hx-post>` con hidden input.
3. **generateAndServe sin transacción** — Dos `db.Exec()` separados para UPDATE download + INSERT payment. Fix: envuelto en `Begin()` / `Commit()`.
4. **Webhook payment updates sin transacción** — Múltiples `db.Exec()` sin atomicidad. Fix: envuelto en transacción.
5. **Tailwind CDN con `unsafe-inline`** — Tailwind desde CDN requería `'unsafe-inline'` en CSP. Fix: compilado a `static/tailwind.css` (v4), servido desde `'self'`. CSP limpiada.

6. **`ratelimit.Check()` nunca era llamado** — Existía la función pero ningún handler la invocaba. Fix: agregado al inicio de `Upload` con key `upload:{IP}`, 10 req/h.
7. **Precio hardcodeado en 3 lugares** — `2990` estaba en `cmd/main.go`, `generateAndServe`, y `createFlowPayment`. Fix: leído de `tools.price_clp` en DB, pasado como parámetro a `generateAndServe` y `createFlowPayment`.
8. **Sin límite de documentos gratis** — Cualquier IP podía generar infinitos documentos en dev mode. Fix: `trial_tracking` table con 30 docs/IP/30 días, `getTrialInfo()`/`recordTrialUsage()` en Pay handler.
9. **Sin pase mensual** — Solo existía el modelo pay-per-use. Fix: plan `pass` en Pay handler, `recordPass()` whitelista IP por 30 días en `trial_tracking`.
